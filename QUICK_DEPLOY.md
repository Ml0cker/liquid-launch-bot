# Quick Deployment Guide

## Step 1: Prepare GitHub Repository

```bash
# Initialize git repo (if not already done)
git init
git add .
git commit -m "Initial commit: Liquid Launch Bot"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/liquid-launch-bot.git
git push -u origin main
```

## Step 2: Add GitHub Secrets

### Option A: Using GitHub CLI (Recommended)

```bash
# Install GitHub CLI: https://cli.github.com/

# Login to GitHub
gh auth login

# Add environment secrets from .env
gh secret set BASE_RPC_URL -b "https://mainnet.base.org" -R YOUR_USERNAME/liquid-launch-bot
gh secret set LIQUID_FACTORY_ADDRESS -b "0x04f1a284168743759be6554f607a10cebdb77760" -R YOUR_USERNAME/liquid-launch-bot
gh secret set START_BLOCK -b "42266500" -R YOUR_USERNAME/liquid-launch-bot
gh secret set TELEGRAM_BOT_TOKEN -b "8546165371:AAFXXRtj42oQwfaKgMwgubhWI-_8jfryb1E" -R YOUR_USERNAME/liquid-launch-bot
gh secret set TELEGRAM_CHAT_ID -b "@Liquid_tokens" -R YOUR_USERNAME/liquid-launch-bot
gh secret set IPFS_GATEWAY -b "https://ipfs.io/ipfs/" -R YOUR_USERNAME/liquid-launch-bot
gh secret set POLL_INTERVAL -b "12" -R YOUR_USERNAME/liquid-launch-bot

# Add server secrets
gh secret set SERVER_HOST -b "YOUR_SERVER_IP" -R YOUR_USERNAME/liquid-launch-bot
gh secret set SERVER_USER -b "ubuntu" -R YOUR_USERNAME/liquid-launch-bot
gh secret set SERVER_SSH_KEY -b "$(cat ~/.ssh/github_deploy)" -R YOUR_USERNAME/liquid-launch-bot
```

### Option B: Manual via GitHub Web

1. Go to: `https://github.com/YOUR_USERNAME/liquid-launch-bot/settings/secrets/actions`
2. Click "New repository secret"
3. Add each secret:
   - `BASE_RPC_URL` = `https://mainnet.base.org`
   - `LIQUID_FACTORY_ADDRESS` = `0x04f1a284168743759be6554f607a10cebdb77760`
   - `START_BLOCK` = `42266500`
   - `TELEGRAM_BOT_TOKEN` = `8546165371:AAFXXRtj42oQwfaKgMwgubhWI-_8jfryb1E`
   - `TELEGRAM_CHAT_ID` = `@Liquid_tokens`
   - `IPFS_GATEWAY` = `https://ipfs.io/ipfs/`
   - `POLL_INTERVAL` = `12`
   - `SERVER_HOST` = Your server IP/domain
   - `SERVER_USER` = SSH username
   - `SERVER_SSH_KEY` = Private SSH key content

## Step 3: Setup Server

SSH into your server and run:

```bash
# Update system
sudo apt-get update && sudo apt-get upgrade -y

# Install Docker and Docker Compose
sudo apt-get install -y docker.io docker-compose git curl

# Add user to docker group
sudo usermod -aG docker $USER
newgrp docker

# Create project directory
sudo mkdir -p /app/liquid-launch-bot
cd /app/liquid-launch-bot

# Clone repository
sudo git clone https://github.com/YOUR_USERNAME/liquid-launch-bot.git .
sudo chown -R $USER:$USER /app/liquid-launch-bot

# Create .env file (will be auto-generated by GitHub Actions)
touch .env

# Create data directory for database
mkdir -p data

# Test Docker setup
docker --version
docker-compose --version
```

## Step 4: Generate SSH Key for GitHub Actions

On your local machine:

```bash
# Generate SSH key
ssh-keygen -t rsa -b 4096 -f ~/.ssh/github_deploy -N ""

# Copy public key to server
ssh-copy-id -i ~/.ssh/github_deploy.pub user@server_ip

# Test connection
ssh -i ~/.ssh/github_deploy user@server_ip "echo 'SSH connection successful'"

# Get private key content for GitHub secret
cat ~/.ssh/github_deploy
# Copy entire output to SERVER_SSH_KEY secret
```

## Step 5: Trigger Deployment

### Option A: Automatic (Push to main)

```bash
git push origin main
# GitHub Actions will automatically deploy
```

### Option B: Manual (Workflow Dispatch)

1. Go to: `https://github.com/YOUR_USERNAME/liquid-launch-bot/actions`
2. Click "Deploy Liquid Launch Bot"
3. Click "Run workflow"

## Step 6: Monitor Deployment

```bash

# https://github.com/YOUR_USERNAME/liquid-launch-bot/actions

# SSH into server and check bot status
ssh user@server_ip
```

## Verification

After deployment, verify bot is running:

```bash
# Check container status
docker-compose ps
# Should show: liquiddeploys-bot-1 ... Up

# Check logs for successful startup
docker-compose logs bot | grep "Starting Liquid Protocol monitor"

# Check database
docker-compose exec -T bot python3 -c "
import sqlite3
conn = sqlite3.connect('/app/data/tokens.db')
cursor = conn.cursor()
cursor.execute('SELECT COUNT(*) FROM processed_tokens')
print(f'Tokens in database: {cursor.fetchone()[0]}')
conn.close()
"

# Check Telegram channel for test message
# Should see: "✅ Liquid Protocol Launch Bot is running!"
```

## Troubleshooting

### Deployment fails with "Permission denied"
```bash
# Fix SSH key permissions on server
chmod 700 ~/.ssh
chmod 600 ~/.ssh/authorized_keys
```

### Docker not found on server
```bash
# Reinstall Docker
sudo apt-get remove docker.io docker-compose
sudo apt-get install -y docker.io docker-compose
sudo usermod -aG docker $USER
```

### Bot not sending Telegram messages
- Check `TELEGRAM_BOT_TOKEN` is correct
- Check `TELEGRAM_CHAT_ID` format (@channel or -100ID)
- Verify bot has admin permissions in channel

### Database not persisting
```bash
# Check data directory permissions
ls -la /app/liquid-launch-bot/data/
# Should show tokens.db file

# Verify volume mount in docker-compose.yml
docker-compose config | grep -A 5 volumes
```

